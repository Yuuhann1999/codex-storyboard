# Product

## Register

product

## Users

短视频创作者在 Codex 侧边栏中规划口播、画面和生成素材。他们需要在同一张表内快速检查镜头结构，并让 Codex 自动创建或修改镜头。

## Product Purpose

把脚本、镜头描述、素材生成方式和最终预览集中到一个本地分镜工作台。成功标准是信息清晰、编辑直接、Codex 写入后立即可见。

## Video Provider Capability

- 项目可以设置默认视频厂商、模型、分辨率、统一视觉风格和主角描述。
- 单个镜头可以继承项目模型或显式覆盖；界面只提供该模型真实支持的时长组合。
- API 视频任务由 Codex 领取和执行，而不是由浏览器直接调用厂商。
- 厂商任务 ID 必须持久化，保证 Codex 超时或重启后继续查询，不重复提交。
- 首版支持 MiniMax Hailuo 2.3 文生视频与首帧图生视频；Provider 注册表预留其他厂商。
- 密钥只能进入本机环境变量，不能写入项目、日志、导出文件或 Git。

## Brand Personality

专业、克制、可靠。

## Anti-references

- 不像营销落地页，不使用宣传口号和装饰性大标题。
- 不使用花哨渐变、玻璃拟态或过度圆角。
- 不使用原生下拉菜单造成不可控的弹层位置。
- 不让固定表头遮挡数据。

## Design Principles

1. 表格内容优先，应用标题和状态只占最少空间。
2. 控件使用统一的尺寸、状态和交互语言。
3. 信息密度接近专业剪辑与项目管理工具。
4. 所有自动化结果都必须能在表格中直接验收。
5. 视觉效果服务于可读性，不承担宣传作用。

## Accessibility & Inclusion

保持清晰对比度、可见键盘焦点、完整键盘操作、减少动画偏好支持，并为图标按钮和表单控件提供可访问名称。
